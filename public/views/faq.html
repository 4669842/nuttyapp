1. What is nutty?

Nutty is a browser tool using which you can share your local desktop terminal console with a remote user. It does not matter if you or the remote user is behind firewall.

2. What advantages does it provide over tmux/screen?

tmux/screen works well when two users on the same machine want to share terminals. When you want to share the terminal with a remote user nutty will help.

3. What is nutty's License?

Nutty is licened under GPLv3

4. In the 3rd step of installation why does it require root access?

curl -s https://raw.github.com/krishnasrinivas/nuttyapp/master/public/install.sh | sudo sh
This is a Chrome limitation. Bug-237873 had been raised which has been fixed in Chrome-34. So you do not need root permission to install nutty script for Chrome-34+ (Stable Chrome-34 has not been released yet)

5. Is nutty like ajaxterm?

Not at all. Ajaxterm lets you login to a server using a browser. Nutty is a terminal sharing tool

6. How do I share terminal using nutty?

Follow the install steps. After install click the icon beside the browser bar. It will open a terminal window inside your Chrome browser and also give you a unique URL which you can share with anyone. That person can access your terminal using the URL even if you and him/her are behind firewall.

7. After sharing terminal, does nutty.io server proxy the data between peers?

You can share terminal with peers using either Websocket or WebRTC. If you choose the Websocket option nutty.io server proxies the data between peers. If you choose the WebRTC option the terminal sharing between peers happen with out any proxy, i.e it is entirely peer-to-peer.

8. What is WebRTC

WebRTC enables peer-to-peer real-time communication in the browser. More info here. Nutty makes use of peerjs.com WebRTC library.

9. Is the terminal sharing encrypted?

Yes. Websocket data transfer uses https-TLS encryption. WebRTC data traffic is encrypted by the browser by default.

10. If I record terminal in nutty and upload, does it store passwords?

No. Because passwords are not echoed back to the terminal.

11. What browsers/OSes are supported?

Right now for sharing has to be done using Chrome-29+ on Linux/MacOS/ChromeOS. In future Firefox will be supported for sharing. For viewing, nutty has been tested on Chrome, it is known to work on FF and Safari regardless of the OS.

12. How can Chrome access terminal device? Isn't Chrome sandboxed?

Nutty uses Chrome's Native Messaging API to provide terminal access to the browser. Nutty extension is restricted to the https://nutty.io domain. Hence other websites will not be able to access/misuse it.

13. Does nutty work on mobile devices?

Yes. A "sharee" can view the shared terminal on mobile devices using mobile browsers. Ability to type in commands from mobile devices does not work right now.

14. I would like to run nutty server on a machine inside my private network(or on EC2), is that possible?

Yes. Since Nutty is GPLv3 you can download the server code and host a private nutty server. Here are the instructions:
  1. Install MongoDB
  2. $ curl https://install.meteor.com/ | sh
  3. $ sudo npm install -g meteorite
  4. $ git clone https://github.com/krishnasrinivas/nuttyapp.git
  5. $ cd nuttyapp
  6. $ mrt install
  7. Configure authinfo.json
  8. $ meteor bundle ../bundle.tgz
  9. $ cd ..
  10. $ tar xzvf bundle.tgz
  11. $ export MONGO_URL=mongodb://localhost/nuttyapp
  12. $ export PORT=80
  13. $ export ROOT_URL='http://yourserver.com'
  14. $ sudo node bundle/main.js
authinfo.json (under private/ directory) should look like this: 
        {
            "google": {
                "clientId": "googleoauth-clientid-optional",
                "secret": "googleoauth-secret-optional"
            },
            "aws": {
                "awsid": "s3upload-awsid-optional",
                "awssecret": "s3upload-awssecret-optional"
            },
            "webrtc": {
                "key": "key from peerjs.com - optional - if you need webrtc"
            }
        }
For webrtc, you can run your own peerjs server too. check peerjs.com for details. In case you want to host a private nutty server you need to do addtional configuration in Nutty's extension to use it:
  • open "chrome://extensions" in address bar
  • click "options" in the Nutty extension section
  • Set your new server domain name.
