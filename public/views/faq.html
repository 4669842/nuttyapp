What is nutty?

Nutty is a browser tool using which you can connect to a machine using SSH and share the terminal session with a remote user. It does not matter if you or the remote user is behind firewall.

What is nutty's License?

Nutty is licened under GPLv3

How do I share terminal using nutty?

Login onto an ssh server using username/password (or a private ssh key). It will open a terminal window inside your Chrome browser and also give you a unique URL which you can share with anyone. That person can access your terminal using the URL even if you and him/her are behind firewall.

Where can I report issues with Nutty?

https://github.com/krishnasrinivas/nuttyapp/issues/new

How do I enable SSH server on Mac OSX?

Go to ‘System Preferences’. Under ‘Internet & Networking’ there is a ‘Sharing’ icon. Run that. In the list that appears, check the ‘Remote Login’ option. Or on the command line do: 'systemsetup -setremotelogin on'

Will my SSH password or Private Key be accessed by nutty server?

No. SSH connection is made directly from your Chrome browser to the SSH server (using nutty's Chrome app) Hence nutty.io server will never access your password or Private Key.

Is my SSH credentials stored on nutty server when I save it?

No. If you save by clicking "Save" the SSH credentials it will be stored on your localdisk in the HTML5 filesystem.

If I share terminal and close my browser tab will a remote user still be able to access the terminal?

No.

How do I copy/paste in the terminal?

Copy: Ctrl-C
Paste: Ctrl-Shift-V

After sharing terminal, does nutty.io server proxy the data between peers?

You can share terminal with peers using either Websocket or WebRTC. If you choose the Websocket option nutty.io server proxies the data between peers. If you choose the WebRTC option the terminal sharing between peers happen with out any proxy, i.e it is entirely peer-to-peer.

What is WebRTC?

WebRTC enables peer-to-peer real-time communication in the browser. More info here. Nutty makes use of peerjs.com WebRTC library.

Is the terminal sharing encrypted?

Yes. Websocket data transfer uses https-TLS encryption. WebRTC data traffic is encrypted by the browser by default.

If I record terminal in nutty and upload, does it store passwords?

No. Because passwords are not echoed back to the terminal.

What browsers are supported?

Right now for sharing has to be done using Chrome. In future Firefox will be supported for sharing. For viewing, nutty has been tested on Chrome, it is known to work on FF and Safari.

Does nutty work on mobile devices?

Yes. A "sharee" can view the shared terminal on mobile devices using mobile browsers. Ability to type in commands from mobile devices does not work right now.

I would like to run nutty server on a machine inside my private network(or on EC2), is that possible?

Yes. Since Nutty is GPLv3 you can download the server code and host a private nutty server. Here are the instructions: https://github.com/krishnasrinivas/nuttyapp/blob/master/README.md
